I want to build a new AI-assisted lane-capacity workflow in Freight-DNA for lanes where we already have recurring freight and need to line up consistent carrier capacity.

Core idea:
This is NOT just a one-load outreach tool.
This should function as a lane-capacity operating system:
- identify recurring lanes worth building around,
- surface them as NBA actions,
- help NAMs / AMs / LMs / Directors / Admins build a reusable carrier bench,
- use AI to rank carriers, draft outreach, interpret replies, and recommend follow-up,
- and move us toward more consistent / semi-contracted carrier coverage on recurring freight.

We already have:
- lane/load history in Freight-DNA
- an uploaded Excel carrier list in the platform (carrier names / attributes, but generally no emails)
- an existing NBA framework
- an AI integration pattern we can use

Important constraint:
The uploaded Excel carrier list should be treated as a one-time seed into Freight-DNA.
After import, Freight-DNA’s own Carriers table becomes the source of truth.
Do NOT try to keep the original Excel in sync.

========================================
3) RECURRING LANE IDENTIFICATION
========================================

I want Freight-DNA to identify recurring “in-house” lanes worth building carrier coverage around.

Base rule:
- Look at the last 4 full weeks
- A lane qualifies if:
  - we moved >= 2 loads/week
  - in at least 3 of the last 4 weeks
- Exclude lanes where hasPreferredCarrierProgram = true

Please make these thresholds configurable so we can adjust later:
- min loads per week
- lookback window
- number of qualifying weeks

Lane context should consider:
- account
- origin
- destination
- equipment
- recurring pickup / delivery pattern if available
- average weekly volume
- margin / service volatility if available

========================================
5) OUTREACH MODES
========================================

This is a critical requirement:
When sending outreach, we need flexibility so we can cast a wide net.

Example:
If we have a load picking up Thursday, but a carrier can’t help until next week, we still want that carrier in our back pocket for future loads on the lane.

So please support TWO outreach modes:

A. Lane-building mode (DEFAULT)
- This is the default mode
- Focus the message on recurring freight on the lane / corridor
- Emphasize:
  - average weekly loads
  - equipment
  - recurring pattern
  - ongoing freight opportunities
- Do NOT make the immediate pickup date the focal point

B. Immediate load + lane mode
- Mention the current load opportunity
- But still frame the outreach as recurring lane freight, not a one-and-done tender

In both modes:
- We want carriers to feel comfortable responding with:
  - available now
  - next week
  - future fit
  - not a fit

========================================
7) AI CAPABILITIES
========================================

I want AI to make this significantly more efficient and powerful, not just write generic emails.

Please include these AI layers:

A. AI lane scoring
Score recurring lanes by:
- consistency of weekly volume
- average loads/week
- margin
- volatility
- strategic account importance
- lack of preferred carrier program
- service risk from ad-hoc coverage

Use this to help prioritize which lane-capacity NBA cards show up first.

B. AI carrier fit scoring
For each carrier/lane pair, generate:
- fitScore
- fitReason

Inputs can include:
- exact lane history
- similar lane history
- equipment fit
- geographic fit
- recency
- service performance
- margin contribution
- any notes if available

Output:
- ranked list of carriers
- short human-readable reason like:
  - “Ran similar TX → IL dry van freight 8 times in last 60 days with strong service”
  - “No direct history, but strong Midwest reefer fit and recent nearby activity”

C. AI email generation
Generate lane-based outreach emails, not singular-load tender blasts.

Requirements:
- adapt tone depending on whether the carrier is:
  - known to us / history carrier
  - new prospect
- allow flexible response:
  - now
  - next week
  - future loads
- voice should match our Value Truck AM playbook:
  - fast
  - solution-first
  - confident
  - relationship-building
  - not robotic

D. AI reply interpretation
When replies come back, use AI to classify the reply into:
- available_now
- available_next_week
- future_interest
- not_fit
- needs_follow_up

Also generate:
- aiSummary of the reply
- suggested next step

E. AI follow-up suggestions
For each lane, recommend next actions such as:
- follow up with carriers who said next week
- re-engage future-interest carriers in 5 days
- stop prioritizing no-response carriers after X attempts
- promote strong recurring responders into preferred carrier review
