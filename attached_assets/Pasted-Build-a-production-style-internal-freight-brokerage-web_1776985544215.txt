Build a production-style internal freight brokerage web application with a primary tab called "Customer Quotes".

This tab must keep all of the original intelligence and analytics functionality, but the experience should now be built around an extremely fast, intuitive, top-of-page "Spot Quote Search" workflow.

This is not a mandatory quote-entry system for all reps. It is a shared visibility and intelligence tool that uses the data we already have from emails, quote requests, won/lost outcomes, load history, and carrier payment data.

The main job of the Customer Quotes tab:
A user should be able to describe a shipment quickly the way a spot quote rep thinks about it, hit search, and instantly pull back everything we know about that lane and related history:
- what we have quoted
- what we won or lost
- what we paid the carrier when we won
- which customers asked for similar freight
- how recent the data is
- how consistent or inconsistent our quoting has been
- whether this is freight we actually want

- same destination region
- same mode

- expiring quotes
