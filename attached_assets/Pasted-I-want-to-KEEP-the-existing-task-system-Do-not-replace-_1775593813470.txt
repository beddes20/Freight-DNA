I want to KEEP the existing task system.

Do not replace tasks with commitments or with NBA actions.

Instead, I want Freight-DNA to support 3 separate but connected action types:

1. NBA = system-generated next best action
- generated by rules/data
- remains the system’s recommended action
- should never be overwritten silently by a human

2. Commitments = weekly coaching commitments
- used in team meetings and 1:1s
- one meaningful growth commitment per rep per week
- tied to account + lever + deadline
- reviewed the following week

3. Tasks = manual assigned work
- used for leadership push-down, delegated execution, and cross-functional action
- examples:
  - Director/NAM assigns a less experienced rep a specific action
  - LM gets assigned a won lane to go source carriers for
  - manager assigns a follow-up item from monthly review
- keep this system intact

NEW FEATURE TO ADD:

Behavior:
- Director/NAM/manager can manually assign a specific action to a rep
- this is NOT the same as the NBA
- do not rewrite the NBA recommendation itself
- instead, surface the manager-pushed action above the NBA as the current leadership priority
