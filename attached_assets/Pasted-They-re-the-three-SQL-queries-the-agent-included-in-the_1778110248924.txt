They’re the three SQL queries the agent included in the dev‑checkpoint playbook. You don’t need to write them yourself; they’ll run them after you roll dev back. But here’s what they are and what they do, in plain terms.

Query (a) – “How many contacts exist at this checkpoint?”
sql
-- (a) Total contacts in the org at the time of the checkpoint
SELECT COUNT(*) AS contacts_total
FROM contacts
WHERE company_id IN (
  SELECT id FROM companies
  WHERE organization_id = 'da3ed822-8846-4435-bb13-3cc4bf26f71d'
);
This just counts how many contact rows exist in your org at that point in time. If it returns hundreds instead of ~17, you know that checkpoint DB still has the old contact universe.

Query (b) – “Show me Armstrong contacts with their rich fields”
sql
-- (b) Specifically — Armstrong contacts with their rich fields
SELECT c.name, c.email, c.title, c.phone, c.lanes, c.regions,
       c.freight_spend, c.spot_bidding_process, c.notes,
       c.interests, c.relationship_base, c.next_steps, c.created_at
FROM contacts c
JOIN companies comp ON comp.id = c.company_id
WHERE comp.organization_id = 'da3ed822-8846-4435-bb13-3cc4bf26f71d'
  AND lower(comp.name) LIKE '%armstrong%'
ORDER BY c.name;
This is the key one for you: it will list any Armstrong contacts at that checkpoint, including the exact fields from your screenshot (lanes, regions, freight spend, spot process, next steps, interests, notes). If Randall and John exist anywhere, this is where they’d show up.

Query (c) – “Just in case they’re under another company name”
sql
-- (c) Zercher / Stone direct hits (in case they were on a different company row)
SELECT id, company_id, name, email, title, lanes, regions, freight_spend, notes
FROM contacts
WHERE lower(email) IN (
    'rzercher@armstrong.com',
    'rlzercher@armstrong.com',
    'jstone@armstrong.com'
)
   OR lower(name) LIKE '%zercher%'
   OR lower(name) LIKE '%john stone%';
This is a safety net: if for some reason they were attached to a mis‑named company instead of “Armstrong,” this query will still find them by name or email.
