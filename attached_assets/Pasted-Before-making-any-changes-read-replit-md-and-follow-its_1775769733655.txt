I want a UI/UX redesign of the Lane Work Queue procurement panel / carrier outreach workspace.
This is primarily a workflow and usability improvement, not a backend ranking rewrite.

Current problem:
The right-side lane drawer feels cramped, overloaded, and not user-friendly enough.
Too many controls compete for space at once:
- lane summary
- assignment controls
- tabs
- carrier filters
- ranked carrier list
- bench/import/follow-up/history navigation
- outreach actions
- email editing near the bottom

The result is that the workflow feels compressed and it is hard to comfortably:
- evaluate carriers
- understand why they rank
- select the right carriers
- edit outreach emails
- move quickly through procurement

Main objective:
Turn the current lane drawer into a cleaner, more efficient, more powerful procurement workspace.

The goal is to help a rep do this flow with less friction:
1. understand the lane
2. review ranked carriers
3. select carriers confidently
4. review/edit outreach
5. send/log outreach
6. move to follow-up/history as needed

This is a frontend / UX-focused task. Reuse existing backend capabilities wherever possible.

What I want conceptually:
The current panel tries to do too much in one narrow space.
I want a more intentional workflow with better information hierarchy and better working space for email editing.

High-level redesign direction:
A. Make the workspace feel less cramped
- Increase the usable width of the right-side workspace if possible
- If a standard drawer is too narrow, convert it into a wider slide-over / workspace panel
- The user should be able to actually work inside it, not just inspect data

B. Separate the workflow into clearer steps or modes
I do NOT want ranking, filtering, selecting, and editing email all fighting for the same vertical space.
Reorganize the right-side experience into a clearer step-based flow, such as:

1. Carriers
2. Message
3. Follow-up
4. History

You can use tabs, segmented steps, or another compact pattern, but the experience should clearly separate:
- carrier discovery / ranking
- outreach drafting / editing
- follow-up tracking
- historical review

C. Keep the lane header compact and useful
The top of the panel should show:
- lane name / origin → destination
- confidence / lane score / cadence / weeks active / contacts reached
- owner / assignment controls
- outreach mode

But this header should be tighter and cleaner than it is now.
It should feel like a concise operating summary, not a dense stack of UI.

D. Make carrier selection the primary first step
The default step should focus on choosing carriers.
That means:
- ranked list is the main content area
- filters are present but not visually overwhelming
- “why this carrier” is clearer
- selection is fast and obvious
- the cards/lists feel easier to scan

Specific carrier-list UX improvements I want:
- clearer badge for ranking tier, e.g. Exact history / Nearby lane / Same corridor
- simpler supporting reasons, e.g. “53 nearby loads”, “Last used Mar 2026”, “Has email”, “Recently active”
- better visual distinction between score, badge, contactability, and quick actions
- faster multi-select behavior
- cleaner density for scanning 10–20 carriers quickly

E. Reduce filter clutter
Right now the chips/filters feel noisy.
Refactor filters into a cleaner pattern:
- keep the most important quick filters visible
- move secondary filters into a collapsible “Refine list” area, dropdown, or popover
- do not make the top of the carrier list feel like a wall of controls

Visible by default should only be the highest-value controls.
Everything else can be tucked under an advanced/refine affordance.

F. Make email editing a real workspace, not a cramped bottom section
This is one of the biggest pain points.
I want editing outreach to feel comfortable and deliberate.

Requirements:
- email editing should have its own dedicated step/state/view inside the lane workflow
- give the user more width and height to edit
- selected recipients should still be visible, but not at the expense of the editor
- subject/body should be easy to read and modify
- personalization / preview should be accessible
- the user should be able to confidently review what is about to be sent

If drafts already exist, reuse them.
If AI-generated draft flow already exists, preserve it.
The improvement is about editing and review UX, not changing the drafting backend.

G. Improve action hierarchy
The bottom action area should feel more intentional.
I do not want a confusing footer with disabled buttons that are always present.

Preferred behavior:
- sticky footer
- primary CTA changes based on current step
- examples:
  - in Carriers step: “Review Message (12 selected)”
  - in Message step: “Generate Drafts” or “Send Outreach”
  - in Follow-up step: relevant next action
- secondary actions should remain available but visually secondary

The user should always know the next best action.

H. Improve Bench / Import / Follow-up / History organization
These are useful, but they currently add to the clutter.
I want a cleaner information architecture.

Possible direction:
- primary workflow tabs: Carriers / Message / Follow-up / History
- move Bench and Import into secondary navigation inside the Carriers step, or as sub-tabs
- do not let low-frequency workflows crowd the main ranking experience

Use judgment here, but optimize for the most common procurement workflow first.

I. Preserve existing product capabilities
Do NOT regress:
- lane assignment actions
- ranked carrier loading
- exact/nearby/state_pair ranking badges and explanations
- bench functionality
- import functionality
- draft outreach generation
- send/log outreach
- history display
- follow-up state
- any existing tests / guardrails

This is a UX/workflow redesign on top of the existing system, not a rewrite of core procurement behavior.

J. Add usability polish
Please improve details like:
- spacing and grouping
- clearer empty states
- better section labels
- more usable scroll behavior
- stronger sticky elements where appropriate
- cleaner disabled/loading states
- easier-to-read card composition
- mobile/responsive behavior if relevant, but desktop workflow is the priority

K. Implementation style
Please do this thoughtfully, not as random spacing tweaks.
Start by identifying:
1. the current panel component structure
2. which component owns the drawer/workspace
3. which component renders the carrier list
4. which component renders the draft editor / drafts tab
5. how tabs and footer actions are currently organized

Then implement a coherent redesign with minimal unnecessary churn.

L. Preferred design principles
Optimize for:
- speed
- clarity
- confidence
- less cognitive load
- better use of space
- obvious next action
- operational power, not decorative redesign

This should feel like a sharper brokerage operating tool, not a prettier but fluffier UI.

M. Deliverables
I want:
1. A short summary of the redesign
2. Exact files changed
3. The new workflow / information hierarchy
4. Notes on what was intentionally moved, collapsed, widened, or split into steps
5. Confirmation that existing procurement capabilities still work
6. Any tests updated or added
7. Screenshots or a clear before/after description of the new lane workspace

N. Success criteria
The redesign is successful if:
- the lane workspace feels less cramped
- carrier ranking is easier to scan and act on
- filters feel cleaner and less noisy
- email editing feels materially easier
- the next action is always obvious
- the tool feels more like a true procurement workspace than a crowded side panel

Important:
Do not spend effort on backend ranking changes unless a small frontend-facing data shape change is absolutely needed.
Focus on the workflow, layout, hierarchy, and usability.

Suggested first step:
Read the full current Lane Work Queue lane drawer / CarrierOutreachPanel implementation and propose the exact component-level redesign plan before coding.
Then implement the redesign cleanly.
