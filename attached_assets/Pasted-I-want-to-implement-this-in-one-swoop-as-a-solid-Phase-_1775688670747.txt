Goal:
Turn the Lane Work Queue / lane detail drawer into a true carrier procurement workflow:
- smarter carrier recommendations
- AI-generated procurement/sourcing emails
- ability to send those emails directly from the platform
- logging so the system gets smarter over time

Current problem:
The drawer is a good start, but “Generate Carrier Outreach” is not completing the workflow. I need the user to be able to identify recommended carriers, generate outreach, edit it, and send it from inside the product.

Important product rule:
The platform should not just show carriers in the region.
It should learn from exact lane history, similar lane history, customer history, equipment history, and outreach outcomes so this becomes smarter over time and that data can be reused elsewhere.

Please implement the lowest-risk but complete Phase 1 version.

4. AI-GENERATED PROCUREMENT / SOURCING EMAILS
==================================================

I want AI-generated general procurement emails that sound like:
- we get this lane regularly
- here are the lane details
- we are looking for a carrier partner who can support this lane consistently

Tone:
- professional
- concise
- carrier-friendly
- partnership-oriented
- not overly polished or robotic

The draft should intelligently use available lane data:
- origin
- destination
- equipment
- avg loads/week
- any consistency signal like weeks active / recurring status
- any relevant note if available

Direction for draft content:
- tell the carrier we move this lane consistently / regularly
- state the lane details
- say we are looking to build consistent carrier support / partnership
- ask if they are interested in discussing availability / coverage / rates
- keep it practical and sendable

Do not hardcode one static template only.
Generate from lane data, but include a fallback template if AI generation fails.
