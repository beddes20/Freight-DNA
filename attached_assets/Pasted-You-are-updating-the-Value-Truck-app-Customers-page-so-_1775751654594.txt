You are updating the Value Truck app Customers page so the Momentum Score / Account Growth Score is visible directly on each customer card in the customer grid.

This is a focused UI enhancement, not a redesign.

Goal
Show the account’s Momentum Score prominently on each customer card in the Customers list/grid view, in the top-right area of the card header (the open space near the phone / add-contact / chevron actions).

Why
Right now users have to infer account health from partial signals or open the account to understand momentum. The score should be visible at scan speed so reps and leaders can quickly identify which accounts are healthy, slipping, or need action.

Placement
- Add the score in the card header area, top-right region of each customer card.
- Keep the existing action icons intact:
  - phone / log touch
  - add contact
  - open chevron
- Do not crowd or break those actions.
- The score should feel like part of the card’s key summary, not a footer stat.

Preferred UI
Render a compact pill / badge / chip such as:
- Momentum 82
or
- Score 82
or
- 82 / 100

Preferred behavior:
- Color-coded by score band.
- Easy to scan.
- Readable without opening the account.
- Consistent across all cards.

Use these score bands unless a shared constant already exists:
- 76–100 = Primed to Grow
- 51–75 = Growth Ready
- 26–50 = Stable / Needs Attention
- 0–25 = At Risk

Suggested visual treatment
- Primed to Grow: green
- Growth Ready: blue or teal
- Stable / Needs Attention: amber
- At Risk: red

If the app already has approved semantic colors/tokens, reuse them instead of inventing new ones.

Data behavior
- Use the existing account growth score / momentum score already computed by the app.
- Do not create a fake placeholder score.
- If the score is already available in the customer list payload, render it directly.
- If the score is not included in the customer list payload, extend the API/selector/query so the card gets the current score efficiently.
- Avoid N+1 fetches from the card component.

Fallback behavior
If a customer does not yet have enough data to compute a score:
- show a subtle neutral badge like “No Score”
or
- show “Score --”
Use the product’s preferred empty-state language if there is already a convention.

Interaction
- On hover or tooltip (if lightweight and already consistent with the app), show a short label such as:
  - Account Growth Score
  - Momentum Score
  - Primed to Grow / At Risk
- If a score explanation drawer/popover already exists elsewhere, reuse it if easy.
- Do not build a large new details modal in this task.

Layout requirements
- Preserve card alignment and responsiveness.
- The score must work cleanly across:
  - customers with owner name shown
  - customers with lane warning badges
  - customers with financial data
  - customers with no financial data
- Do not let the score overlap icons, wrap awkwardly, or cause card height jitter.

Implementation guidance
1. Find the customer card component used in the Customers page grid.
2. Identify where the existing account growth / momentum score is currently computed or fetched.
3. Wire that score into the customer card data model.
4. Add a reusable score badge component if helpful.
5. Use shared score-band helper logic if it already exists; otherwise create a small helper:
   - getAccountScoreBand(score)
   - getAccountScoreColor(score)
   - getAccountScoreLabel(score)

Copy guidance
Preferred visible text options, in order:
- Momentum 82
- Score 82
- 82

If space is tight, use the shortest form on mobile.
If helpful, tooltip can say:
- Account Growth Score: 82
- Growth Ready
- At Risk

Technical constraints
- Keep this scoped to the Customers list/grid page.
- Do not redesign the whole card.
- Do not move primary actions unless absolutely necessary.
- Reuse the existing score source and business logic.
- Avoid duplicate score-band logic in multiple files.

Acceptance criteria
- Every customer card shows the current momentum/account growth score when available.
- The score appears in the top-right card header area.
- Existing action icons still work and layout remains clean.
- Score color reflects band.
- Cards with no score show a graceful fallback.
- No per-card API fetches.
- Mobile and desktop layouts both render correctly.

Tests
Add/update tests for:
- score renders when present
- correct band/color/label for representative scores (e.g. 18, 42, 68, 91)
- fallback renders when score missing
- header layout still renders actions correctly
- customer list query includes score if it did not before

Output
