I want a new task for the Available Freight Cockpit based on the current live page.

Please design and implement an Account Manager filter and improve the default pickup-recency behavior so the main list shows more actionable freight by default.

What I’m seeing

On the current cockpit, I’m still seeing rows with badges like “Pickup was 2d ago” in the main working list. That creates noise for reps because these are usually not the loads they should be prioritizing first.

What I want
1) Add an Account Manager filter

Add a new filter control for Account Manager near the existing customer / queue / sort / group filters.

Requirements:

Support:

All account managers (default)

My accounts

Specific account manager selection

Optional Unassigned / no account manager bucket if that exists in the data

Make sure the filter works consistently with existing filters like customer, queue status, pickup recency, and saved views.

Persist the filter in the URL / saved view state if that pattern already exists on this page.

Make the label intuitive for reps and managers — if “Account Manager” is not the best UI label, propose the best wording but keep the business meaning the same.

2) Tighten default pickup-recency behavior

I do not want the default working view to show freight that was supposed to pick up yesterday, two days ago, etc. unless there is a strong reason it is still actionable.

Please propose and implement a better default recency rule for the main queue.

Preferred direction:

Default main view should prioritize upcoming + truly current opportunities.

Loads whose pickup is already in the past should be excluded by default from the main actionable queue, OR moved into a clearly-labeled stale / expired / past-pickup bucket.

If we still need visibility into them, give users an explicit way to reveal them, such as:

a filter like Pickup: upcoming only / include overdue / overdue only

or a chip like Show past-pickup

or a hidden-count explainer similar to the current AF empty-state semantics.

3) Keep nuance — don’t hide real exceptions blindly

I do understand there are edge cases where a load that was supposed to pick up recently may still be actionable.

So I do not want a blunt hard-delete from visibility.
I want a rule that distinguishes:

pickup in the future,

pickup today,

pickup overdue but still plausibly actionable,

pickup stale / expired / should not clutter the main queue.

Please recommend the best operational rule here, not just a simplistic date filter.

For example, I’d like your recommendation on something like:

Default: today + future only

Soft overdue: yesterday or within X hours, only if still open and not covered

Stale: older than that, hidden by default but recoverable via filter/chip

If you think a better policy exists, propose it and explain why.

4) UX recommendations

Please also recommend 3–5 improvements to this filter bar / cockpit UX based on the current screen.

I’m especially interested in:

reducing noise,

making ownership clearer,

making stale freight easier to suppress,

and helping reps work the highest-value loads first.

Ideas you can evaluate:

account manager + rep owner as separate filters,

an explicit actionable only toggle,

better saved-view support,

a stale / overdue count chip,

default sort improvements,

or a smarter “My freight today” definition.

5) Deliverable

Please come back with:

your recommended product behavior,

the exact filters / states to add,

implementation plan,

any schema/API/query changes required,

and how you’ll guardrail-test the behavior so we don’t regress the Available Freight cockpit.

Important:

Optimize for day-to-day rep usability, not maximum theoretical visibility.

The default view should answer: what should I work right now?

Keep managers able to inspect stale / overdue freight intentionally, but don’t let that noise dominate the main list.
