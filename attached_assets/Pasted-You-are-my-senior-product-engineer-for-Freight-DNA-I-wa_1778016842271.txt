SUB-TASK 1 — First-touch tender → freight opportunity creation
Objective:
If an inbound customer email is clearly a first-touch tender / load offer (not just a quote request), create a freight opportunity record automatically in a safe status like pending_approval or needs_review, linked back to the source email.

Requirements:
- Reuse existing email ingestion/classification infrastructure where possible.
- Detect obvious tender/load-offer signals from email text and/or existing extracted signals.
- Create a freight_opportunities row with:
  - orgId
  - source='email'
  - sourceReference=providerMessageId (or equivalent)
  - customer/account if known
  - lane/equipment/date placeholders if partially known
  - a safe initial status such as pending_approval or needs_review
- Do NOT auto-book or auto-dispatch.
- Add a source badge / explanation in the freight opportunity UI so the user knows this came from email.
- Preserve idempotency: the same message cannot create duplicate freight opportunities.

Acceptance test:
A clear inbound tender email like “Please cover ATL to DAL, reefer, pickup Tuesday, PO attached” creates exactly one linked freight opportunity within the existing ingestion flow.

  - delivery city/state

- Show these hints in the Needs Routing drawer or equivalent review UI.

Acceptance test:

Improve attribution for Gmail/Outlook/free-mail senders so meaningful customer emails are less likely to fall into Unknown/Unlinked buckets.
