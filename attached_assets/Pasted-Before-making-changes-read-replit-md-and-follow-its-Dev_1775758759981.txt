Task:
Unify procurement assignment workflow so reps handle awarded-lane assignments and Lane Work Queue assignments in one identical operational flow.

Business goal:
Today, procurement work can originate from multiple places, but the assigned rep should experience one consistent workflow:
- they have one obvious place in the portal to see everything assigned to them
- each assigned item is a lane-centric procurement task
- clicking the assignment takes them directly into the lane procurement workspace
- from there they can immediately review carriers, draft outreach, and email all targeted carriers from the existing procurement/outreach UI

What already exists and must be reused:
- lane tasks already exist
- assignment can already create a work assigned lane task
- resolving the lane can auto-complete lane-procurement tasks
- Lane Work Queue already has the lane + right-side procurement/outreach panel
- ranked carrier suggestions already exist
- draft outreach and send outreach endpoints already exist
- CarrierOutreachPanel already has Drafts / send / log-only behavior
- outreach history already logs to lane/carrier history

What to build:
A. One unified assigned-work surface
Create a clear portal destination for the assignee, named something like:
- My Procurement
or
- Assigned Lanes

This page should:
- default to assigned-to-me
- show procurement assignments from both award-originated work and Lane Work Queue-originated work
- not force the rep to care where the task came from
- present the work in one uniform lane-centric list

B. One shared assignment model
Ensure both sources feed the same lane-procurement task/work item model.
Each item should at minimum expose:
- lane id
- task/work id
- source type (award or work queue) for reporting, but not as a primary UX distinction
- customer/account name
- origin / destination
- equipment
- assigned user
- assigned at
- age / SLA indicator
- status (new, in progress, waiting, completed, etc.)
- deep link target into the procurement workspace

If there are currently separate creation paths, normalize them so the assignee sees one unified output.

C. One-click open into the procurement workspace
When the rep clicks an assigned lane, take them directly to the lane procurement screen, not to a generic task detail dead-end.
Target behavior:
- opens the lane workspace
- opens the right-side procurement/outreach panel automatically if possible
- makes it easy to immediately:
  - review ranked carriers
  - understand why top carriers are ranked
  - select carriers
  - draft outreach
  - email all selected carriers at once using the existing flow
  - review history / prior outreach

D. Make the process operationally uniform
The exact same sequence should be true whether the work started from:
- an award
- the lane work queue

Desired rep flow:
1. Open My Procurement
2. See all assigned lanes in one list
3. Click a lane
4. Land directly in the procurement workspace for that lane
5. Review ranked carriers
6. Draft and send outreach to selected carriers
7. See history and status
8. Update / resolve the task

E. UX requirements
The My Procurement list should be fast and simple.
Recommended columns / fields:
- Customer
- Origin → Destination
- Equipment
- Status
- Priority / aging
- Assigned at
- Source (small secondary badge only if useful)
- CTA: Open

Recommended filters:
- Assigned to me
- New
- In progress
- Waiting
- Completed
- Overdue / aging
- Optional source filter for internal admin visibility only

Do not overcomplicate this page.
This is an inbox / queue for procurement reps, not a reporting dashboard.

F. Routing / deep-linking
Implement a durable route pattern so a task row can deep-link directly into the lane procurement workspace.
Examples:
- /lanes/work-queue?laneId=...
- /lanes/work-queue/:laneId
- optional query param to auto-open procurement panel or select the task

Expected behavior:
- if lane exists and rep has access, it opens directly into the actionable workspace
- if task is clicked from My Procurement, the correct lane should be selected automatically
- if possible, preserve a query param or state to auto-open the outreach/procurement drawer

G. Status / workflow behavior
Task state should reflect actual work progression.
Recommended statuses:
- New
- In Progress
- Waiting / Follow-up
- Completed

Suggested rules:
- assignment creates New
- opening or interacting can move to In Progress
- outreach sent can support Waiting / Follow-up state
- lane resolved / procurement completed can auto-complete the task
- avoid duplicate tasks for the same lane + assignee + active procurement workflow unless there is a real business reason

H. Preserve existing outreach behavior
Do not regress:
- ranked carrier suggestions
- draft outreach flow
- send outreach flow
- outreach history
- Activity tab
- CarrierOutreachPanel functionality

This task is primarily about workflow unification, task surfacing, and direct navigation into the existing procurement workspace.

Implementation expectations:
- reuse current task and lane workflow primitives where possible
- do not create a second procurement UI if the existing lane workspace already supports the work
- avoid introducing a parallel assignment system
- keep source reporting in the data model, but keep UX focused on “here is your assigned lane work”
