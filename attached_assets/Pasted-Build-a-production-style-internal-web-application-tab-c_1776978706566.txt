Build a production-style internal web application tab called "Customer Quotes" for a freight brokerage team.

This app is for internal visibility and analytics around customer quote requests, spot quoting activity, outcomes, and lane performance. The goal is NOT to require every rep to use a new pricing form. Instead, the app should create transparency and shared intelligence from data we already have, especially quote requests from emails, historical quote activity, won/lost results, and carrier cost data.

Use a polished, data-dense, dark-theme design with:
- Background: #0A0A0A
- Accent: #FFC333
- Text: white and muted grays
- Font: Inter or a similar clean sans-serif
- Tight 4px corner radius
- Minimal decorative styling
- High usability for operational users
- Keyboard-friendly controls
- Strong table UX
- Sticky filters
- Responsive layout, but optimized for desktop first

Build this as a full-stack Replit app with:
- frontend for dashboard UI
- backend API routes
- database schema for quote opportunities and related entities
- seed/demo data so the app is fully usable immediately
- clean architecture so real email/TMS/CRM integrations can be added later

APP NAME / PRIMARY TAB
- The main tab must be called "Customer Quotes"

CORE PRODUCT VISION
Create a centralized lane-opportunity intelligence workspace where users can:
- see what customers are asking us to quote
- see what we quoted
- see if we won or lost
- see what we paid the carrier if we won
- compare performance by customer, lane, rep, time period, and outcome reason
- identify which freight is desirable vs low-quality
- improve quoting speed, consistency, transparency, and margin management

IMPORTANT PRODUCT PRINCIPLE
Do not design this as a mandatory quote-entry tool. Design it as a transparency and intelligence layer first. Assume the data can come from:
- emails
- imported quote records
- load history
- execution data
- carrier payment data
- rep attribution

2. STICKY FILTER BAR
Directly under the header, create a sticky filter row with:
- Customer filter (high priority, searchable dropdown)
- Date range filter
- Equipment filter
- Rep filter
- Outcome status filter
- Outcome reason filter
- Lane search field (search by origin, destination, or combined lane)
- Toggle: show only won
- Toggle: show only active quotes
- Clear all filters action

Show active filters as chips/tags so users always understand what is being filtered.
Customer filtering must be a first-class interaction. When a customer is selected, the whole page should pivot into that customer's view and update all KPIs, charts, and tables. Use visible active filter chips because persistent filter visibility improves clarity and trust in dashboards. [web:53][web:56]

MAIN CONTENT BLOCKS

A. Customer Performance Panel
When a customer is selected, show:
- Customer name
- Total quotes
- Win/loss rate
- Avg quoted amount
- Avg carrier buy
- Avg gross margin
- Avg response time
- Top quoted lanes
- Top loss reasons
- Trend over time

- Which lanes do we win?
- Where do we lose?
- Are margins healthy?

B. Quote Outcome Taxonomy Module
Do not only track won/lost.
Add structured outcome categories:
- Won
- Lost on price
- Lost on service
- Lost on timing / slow response
- Lost to incumbent
- No response / went dark
- Expired
- Won but low margin

Show:
- distribution chart
- trend over time
- breakdown by customer
- breakdown by rep
- breakdown by lane

This module is mandatory because segmented win/loss reason tracking makes analytics actionable. [web:31][web:44]

C. Quote Validity Window Module
Each quote opportunity should include:
- quote sent date
- valid through date
- active / expired / expiring soon state

Show:
- expiring quotes list
- aging buckets
- stale quote count
- active vs expired summary

Use this to help users understand when historical pricing may no longer be reliable.

D. Lane Overlap / Internal Variance Module
Create analytics to show:
- same lane quoted by multiple reps
- quote spread on same/similar lanes
- quote spread by customer
- unusually high internal variance
- inconsistent quoting behavior flags

This should help leadership identify where the team is pricing inconsistently.

E. "Should We Want This Freight?" Scoring Module
Create a lane/customer opportunity score that combines:
- historical win rate
- avg gross margin
- margin volatility
- carrier coverage depth
- repeat frequency
- service quality if available
- strategic customer value

Display score labels such as:
- Pursue Aggressively
- Good Freight
- Selective
- Low Quality / Low Return

This should help users evaluate whether winning more of that freight is desirable, not just possible.

F. Operational Alerts Module
Right column or top-right panel:
- lanes quoted frequently but rarely won
- quotes expiring soon
- customers with rising quote volume
- quotes with low-margin risk
- lanes with high internal variance
- requests waiting too long for quote response
