GOAL
I want a clear summary of everything that changed today so I can decide whether it's safe to deploy.

CONSTRAINTS
- Do NOT make any additional code changes.
- Do NOT run migrations or any destructive commands.
- Just inspect the repo and tell me what changed.

  - backend logic (TS/JS)
  - frontend/UI
  - migrations/schema
  - background jobs/schedulers/scripts

- files touched

- contacts, org charts, or suggestions

For anything you can’t fully infer, list:
- the file
- your best guess of its purpose
- what I should verify manually before deploying

5) Output format
Return your answer in this structure:

1. Summary of today’s changes (bullet list)
2. Low-risk changes
3. Medium-risk changes
