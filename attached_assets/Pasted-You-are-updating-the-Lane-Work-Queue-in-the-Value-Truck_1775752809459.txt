You are updating the Lane Work Queue in the Value Truck app to make lane presentation cleaner and procurement outreach feel more natural, more freight-native, and more useful.

This is an in-place product refinement, not a redesign.

Goals
1. Clean up how lanes are displayed in Lane Work Queue.
2. Improve default AI-generated outreach copy so it sounds like a freight broker, not a stiff template.
3. Reduce repetitive/system-generated phrasing.
4. Make recurring-lane outreach easier to send with less manual cleanup.

Current problems
- Lane text is not normalized/capitalized correctly.
- Some lanes show duplicate state values, e.g. “marietta, pa, pa” or “macon, ga, ga”.
- The default email sounds too formal and robotic.
- “carrier bench” is not good wording and should be removed.
- Average loads are shown as decimals like 5.10 instead of a human-friendly range.
- Copy should use more freight lingo and feel more natural.

Examples of bad current output
- macon, ga, ga → marietta, pa, pa
- averaging 5.10 loads per week
- we value our relationship

Part 1: Lane formatting cleanup

Implement a centralized lane/location formatter used by:
- Lane Work Queue list rows
- Lane Work Queue right-side drawer/header
- AI email generation context
- fallback email templates
- any lane preview text shown in follow-up/history if applicable

Formatting rules
- City and state should be title-cased / uppercased correctly:
  - Macon, GA
  - Marietta, PA
- Never show duplicated state values.
- If city/state parsing produces a duplicate like “Macon, GA, GA”, collapse it to “Macon, GA”.
- Use a single canonical formatter everywhere instead of duplicating string cleanup in multiple components.
- Preserve other useful location info only when it adds value and is clean.

Acceptance examples
- “macon, ga, ga” -> “Macon, GA”
- “marietta, pa, pa” -> “Marietta, PA”
- “marietta, pa” stays “Marietta, PA”

Part 2: Load volume phrasing cleanup

Do not use raw decimals like:
- averaging 5.10 loads per week

Instead, convert weekly load averages into human-friendly language for outreach copy.

Preferred rules
- Round to an integer or small range.
- For recurring lanes, prefer language like:
  - usually 5–7 loads a week
  - around 4–6 a week
  - typically 2–3 a week
  - usually 1–2 a week

Suggested logic
- Use the average as the center.
- Convert to a soft range rather than a decimal exact number.
- Keep it simple and believable.
- If volume is very low, say:
  - a couple loads a week
  - around 1–2 a week
- If volume is higher, still keep it natural:
  - usually 8–10 a week
  - around 10+ a week

Create a helper for this so the same phrasing is reused consistently.

Update the AI prompt and fallback recurring-lane template so outreach sounds:
- less formal

Do not use “carrier bench” anywhere in recurring lane outreach.

Choose based on context, but keep it natural.

Part 4: Better recurring lane email structure

For recurring lane emails, prefer a structure like:

1. Short greeting
2. Quick reason for reaching out
3. Clean lane string
4. Human volume phrasing
5. Direct ask
6. Light close that keeps future freight open

Good structure example
“Hey [carrier/team] —
We’ve got regular freight on [lane] and usually see about [range] a week.
Wanted to see if this lane fits your network and if you’d be interested in covering some of it.
If this week’s tight, no worries — happy to keep you in the mix for upcoming loads.”

Bad structure example
- long intro
- relationship boilerplate
- exact decimals
- vague ask
- “carrier bench” wording
