Freight-DNA — Dashboard Trust Pass (Phase 1.5) — Trending + Margin Freshness Labeling

Context:

Freight-DNA is a high-trust CRM + workflow OS for freight brokerage.

Phase 1.5 freshness work already completed:

Award Health + Coverage Gaps now show stale vs unknown vs healthy-empty correctly.

NBA now shows “Recommendations may be stale” / “Recommendation freshness unavailable” using data-driven freshness from max(nba_cards.created_at).

The next dashboard trust gap is director/admin-facing financial surfaces:

Trending Accounts

Margin Metrics

Current issue:

these surfaces can show real-looking numbers with no visible “as of” signal

if uploads lag, users may read last month as if it were current

even when sync-alert is visible, the charts/cards themselves still do not state what month/data currency they represent

Goal:
Add clear, low-noise freshness labeling to Trending Accounts and Margin Metrics so users can see the actual data recency/source without changing any underlying math.

Guardrails:

No schema changes.

No scheduler changes.

No changes to financial math, trend logic, goals logic, or ranking.

No edits to alias/fuzzy matching logic.

Keep this slice to labeling/trust UI only.

Prefer additive metadata and UI labels over refactoring endpoint payloads broadly.

Do not build the summary-level pipeline health strip in this task.

Scope:

In scope:

server freshness/source labeling for:

GET /api/dashboard/trending-accounts

GET /api/dashboard/margin-metrics

client rendering of:

“as of <month/date>”

freshness / stale / unknown labeling if available

Out of scope:

Award Health / Coverage Gaps / NBA

pipeline health strip

financial formula changes

monthly sync scheduler changes

upload workflow fixes
